# User Memory & RAG — 정책 문서 템플릿 (초안)

**문서 ID**: USER-MEM-RAG-001  
**버전**: 0.1-draft  
**상태**: 법무·PM 검토 전 — 엔지니어링 게이트용 체크리스트

이 파일은 Phase C(C1) 산출의 **초안 템플릿**입니다. 승인본은 Notion/Confluence 등 정책 저장소에 두고, 여기에는 버전·링크만 유지해도 됩니다.

---

## 1. 처리 목적 및 법적 근거

- [ ] 개인화 코칭(운동 맥락·선호·이력 요약)을 위한 처리 목적 명시
- [ ] 적용 법령·근거(예: 개인정보보호법상 동의/이용 목적)와 서비스 약관·개인정보 처리방침과의 정합성

## 2. 수집 항목

- [ ] 저장되는 데이터 유형(요약 텍스트, 청크, 메타데이터 `source`/`topic` 등)
- [ ] 수집하지 않는 항목(실명, 연락처 등 PII 자제) 및 앱·서버측 필터링 원칙

## 3. 동의

- [ ] 최초 수집·동기화 시점의 **명시 동의** UI 카피
- [ ] 동의 철회 시 업로드 중단·삭제 절차와 UI

## 4. 보존 및 파기

- [ ] 보존 기간(예: 미사용 N일, 탈퇴 시 즉시 삭제)
- [ ] 예외 보존(분쟁·법적 의무) 유무 및 기간
- [ ] 자동 삭제 작업 주체(배치/이벤트)와 감사 로그 범위(원문 미기록)

## 5. 삭제권(잊힐 권리)

- [ ] 앱 내 “메모리 초기화” 동작 정의
- [ ] 백엔드 벡터 저장소(Pinecone 등)에서 해당 사용자 네임스페이스 삭제 API 호출 포함
- [ ] 삭제 완료 확인 SLA(최대 지연)

## 6. 비식별화·안전

- [ ] 로그에 사용자 원문·식별자 평문 금지
- [ ] 내부 식별자: 서버 발급 `subject` 및 벡터 네임스페이스는 해시 기반 키(`u_` + 해시 등)
- [ ] 임베딩·벡터 DB 업체(OpenAI, Pinecone 등) DPA·전송 국가·재위탁 고지

## 7. 제3자·국외 이전

- [ ] 서브프로세서 목록 및 변경 통지 방법
- [ ] 국외 이전 시 고지·동의 요건 충족 여부

## 8. 엔지니어링 게이트 (C3 API)

- [ ] 위 항목 법무·PM **승인** 전: `POST /memory/*` 는 스테이징 전용 또는 기능 플래그 OFF
- [ ] 승인 후 프로덕션 롤아웃 체크리스트(모니터링, 쿼터, 사고 대응)
